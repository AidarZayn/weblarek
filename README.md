# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Модели данных
#### Данные:
Интерфейс товара `IProduct`
Интерфейс описывает структуру товара в каталоге.
Поля:

- `id` — уникальный идентификатор товара,
- `title` — название товара,
- `description` — описание товара,
- `image` — ссылка на изображение товара,
- `category` — категория товара,
- `price` — цена товара; может отсутствовать (undefined или null).

Тип платежа `TPayment`
Тип `TPayment` представляет собой строку, описывающую способ оплаты (например, «наличными», «банковской картой»).

Интерфейс покупателя `IBuyer`
Интерфейс описывает данные покупателя при оформлении заказа.
Поля:

- `payment` — способ для оплаты ( наличным расчетом или картой ),
- `address` — адрес доставки покупателя,
- `email` — электронная почта покупателя,
- `phone` — номер телефона покупателя.

Интерфейс GET запроса с сервера `ICatalogFromApi`
Интерфейс нужен, чтобы описать ответ от сервера.

- `total` - общее число карточек из каталога,
- `items` - это сам массив карточек из каталога.

Интерфейс результата оформелния заказа `IOrderResult`
Интерфейс результата оформелния заказа нужен для ответа от сервера, когда пользователь оформил заказ и получил от сервера success промис.
Поля:

- `id`, это ID заказа,
- `total`, это сумма всех оформленных товаров из заказа.

Тип для установки разрешенных методов для работы с сервером `ApiPostMethods`.

Интерфейс для создания серверного класса `IApi`
Интерфейс создан, чтобы задать строгие правила для оформления серверного класса.
Свойства:

- метод get 
    принимает в себя дженерик наследуемого от объекта `T extends object`
    урл, который добавится к урлу апишки
    возвращает типизированный промис, ответ от сервера типизируемый дженериком,
- метод post
    дженерик наследуемого от объекта `T extends object`
    урл, который добавится к урлу апишки
    данные, которые будут являться объектом
    необязательный параметр методов, которые могут быть использованы в запрос
    возвращает типизированный дженериком промис, который должен прийти от сервера.

Интерфейс `ISuccessActions`
Интерфейс описывает действия (хэндлер) после успешного оформления заказа.

Интерфейс `ICardActions`
Интерфейс описывает действия внутри карточки.

Интерфейс `IFormActions`
Интерфейс описывает действия внутри формы оформления заказа.
#### Классы
Класс `Catalog`  
Является классом для хранения товаров.

Внутреннее состояние:  
`products` — массив товаров типа `IProduct[]`, изначально пустой
`selectedProduct` — выбранный товар, возможно отсутствующий (null в начальном состоянии).

Конструктор:  
`constructor()` - инициализирует объект с пустым массивом для товаров и выбранный элемент со значением `null`.

Методы каталога:  
`get catalogData(): IProduct[]` - геттер для получения каталога товаров, геттер возвращает массив товаров или пустой массив типом `IProduct[]`,  
`set catalogData(products: IProduct[])` - сеттер принимает параметр массив товаров с типом `IProduct[]` и заменяет им внутренний массив товаров каталога,  
`get selectedProductData(): IProduct | null` - геттер позволяет вернуть выбранный товар, если он выбран с типом `IProduct`, но если товар отсутствует то вернется `null`,  
`set selectedProductData(product: IProduct)` - сеттер позволяет установить выбранный товар из параметра с типом `IProduct`,  
`findProductById(productId: string): IProduct | null` - данный метод выполняет поиск в массиве товаров по `ID` с помощью передаваемого `ID` с типом `string`. Если товар найден, то мы возвращаем найденый товар с типом `IProduct`, в случае если не нашли товар вернем `null`.  

Класс `Basket`  
Является классом для хранения выбранных товаров.

Внутреннее состояние:  
`basket` - корзина, которая изначально является пустым массивом и протипизирован `IProduct[]`.

Конструктор в данном классе отсутствует.

Методы коризны:  
`get productsBasket(): IProduct[]` - геттер возвращает массив элементов в корзине с типом `IProduct[]`,  
`addItemInBasket(value: IProduct)` - метод принимает в параметр объект типом `IProduct` и добавляет объект в массив для выбранных товаров,  
`deleteProductInBasket(id: string)` - метод принимает в параметр `ID` с типом `string`, чтобы удалить товар из корзины по его `ID`,  
`clearBasket(): void` - метод ничего не возвращает, но очищает массив выбранных товаров,  
`get amountBasket(): number` - метод для расчета всех товаров по его стоимости, также присутствует проверка на `null`, возвращает `number`,  
`get basketLength(): number` - геттер возвращает длину массива корзины с типом `number`,  
`hasProductInBasket(id: string): boolean` - метод принимает параметром `ID` с типом `string`, аргумент `ID` ищется в массиве корзины, метод возвращает `boolean`.  

Класс `Buyer`
Является классом для хранения данных покупателя.

Содержит инициализацию шаблона пустых данных для покупателя.

Внутреннее состояние:  
`buyer` - покупатель, который изначально является шаблоном пустых данных покупателя.

Конструктор:  
`constructor()` - инициализирует объект покупателя с пустыми данными по шаблону.

Методы покупателя:   
`get buyerData(): IBuyer` - геттер возвращает объект данных покупателя с типом `IBuyer`,  
`set buyerData(Buyer: Partial<IBuyer>)` - сеттер, который позволяет устанавливать по отдельности данные, которые прописаны в интерфейсе `IBuyer`,  
`clearBuyerData(): void` - метод который ничего не возвращает, но очищает данные покупателя,  
`validateAddress(): {[key: string]: string}` - метод возвращает объект ошибок, в котором проводится валидация адреса покупателя на пустоту или `null`, в результате если поле пустое или `null`, то полю присваивается текстовое сообщение ошибкой.  
`validateContacts(): {[key: string]: string}` - метод возвращает объект ошибок, в котором проводится валидация электронной почты и номера телефона покупателя на пустоту или `null`, в результате если поле пустое или `null`, то проверяемому полю присваивается текстовое сообщение ошибкой. 

### Слой коммуникации
Класс `ApiModel`  
Является слоем коммуникации между приложением и сервером, он отвечает за получение каталога товаров и оформление заказа пользователя.

Внутреннее состояние:  
`api` — экземпляр объекта, реализующего интерфейс `IApi`, через который `CatalogService` выполняет HTTP‑запросы к серверу.

Конструктор:  
`constructor(api: IApi)` — инициализирует объект `CatalogService`, принимая в параметре экземпляр `IApi` (например, готовый `Api`) и сохраняя его во внутреннее поле `api`. Это позволяет использовать уже реализованные базовые методы `get` и `post`.

Методы класса:  
`getCatalogProducts(): Promise<IProduct[]>` - Метод выполняет запрос к серверу для получения списка товаров каталога.
Внутри вызывается универсальный `get` метод интерфейса `IApi` с путём `/product`.
Метод возвращает `Promise`, который разрешается в объект типа `ICatalogFromApi`.  
`postOrder(reqData: IOrder): Promise<IOrderResult>` - Метод отвечает за оформление заказа пользователем.
Внутри вызывается универсальный `post` метод интерфейса `IApi` с путём `/order` и передачей объекта заказа `reqData` типа `IOrder`.
Метод возвращает `Promise`, который разрешается в объект типа `IOrderResult`, содержащий результат оформления заказа.

Является классом для управления модальным окном, в котором происходит: закрытие и обработка, отображение модального окна.

Внутреннее состояние:
`closeButton` - кнопка для закрытия модального окна.
`modalContent` - контейнер в котором находится контент модального окна.

Методы класса:
`set content(content: HTMLElement | null)` - сеттер, который устанавливает или обнуляет контент в модальное окно.
`open(): void` - метод, который открывает модальное окно, путем добавления класса (modal_active).
`close(): void` - метод, который закрывает модальное окно, путем удаления класса (modal_active) и удалением контента модального окна.

Класс `SuccessView`
Является классом для подтверждения успешного оформления заказа с итоговой суммой.

Внутреннее состояние:
`buttonWindow` - кнопка закрытия окна успешной оплаты;
`descriptionWindow` - отображение итогой суммы окна успешной оплаты;

Методы класса:
`set total(price: number)` - сеттер, который устанавливает итоговую сумму списания.

### Презентеры
Презентеры отвечают за обработку событий веб-приложения в файле Main.ts
Все события перечислены в файле Events.ts

1. `basket:open` — Открытие модального окна корзины.
2. `basketModel:change` — Изменение данных в модели корзины.
3. `basket:orderButtonClick` — Переход к оформлению заказа из корзины.
4. `buyerModel:change` — Изменение данных покупателя в модели.
5. `buyerModel:clearData` — Очистка данных покупателя.
6. `catalogModel:setAll` — Установка списка товаров в каталоге.
7. `catalogModel:setSelected` — Выбор конкретного товара для предпросмотра.
8. `cardCatalog:click` — Клик по карточке товара в каталоге.
9. `cardBasket:delete` — Клик по кнопке удаления товара в корзине.
10. `cardPreview:buttonClick` — Общий клик по кнопке в окне предпросмотра.
11. `formOrder:submit` — Сабмит формы заказа (переход к контактам).
12. `formOrder:changePayment` — Изменение способа оплаты.
13. `formOrder:changeAddress` — Изменение адреса доставки.
14. `formContacts:submit` — Сабмит формы контактов (отправка заказа).
15. `formContacts:changeEmail` — Изменение email в форме контактов.
16. `formContacts:changePhone` — Изменение номера телефона в форме контактов.
17. `success:submit` — Закрытие окна успешного оформления заказа.
18. `modal:close` — Закрытие модального окна.
